# PDV Leve em Python

## Visão Geral

Este projeto é um **PDV (Ponto de Venda)** desenvolvido para operar com **baixo consumo de recursos**, priorizando simplicidade, portabilidade e autonomia.  
Ele foi projetado para funcionar em **qualquer ambiente capaz de executar Python**, sem dependência de infraestrutura pesada ou serviços externos.

O sistema pode ser executado localmente, inclusive em ambientes como **Android (via Termux)**, ou remotamente em uma **VPS de baixo custo**.

## Propósito

O propósito deste projeto é viabilizar um sistema de PDV funcional em cenários de **restrição de hardware, conectividade ou orçamento**, onde soluções tradicionais se tornam inviáveis ou desproporcionais.

A motivação central é permitir que comerciantes tenham controle total do processo de venda, sem dependência de plataformas proprietárias, licenças caras ou hardware específico.

## Objetivo

O objetivo principal é fornecer um **PDV leve, estável e extensível**, capaz de:

- Rodar em máquinas com recursos limitados
- Funcionar em ambientes não convencionais
- Operar localmente ou de forma remota
- Manter simplicidade operacional e técnica
- Facilitar manutenção e adaptação do código

Tudo isso utilizando tecnologias simples, amplamente conhecidas e fáceis de manter.

## Escopo

Este projeto **não tem como objetivo** competir diretamente com suítes comerciais completas de PDV ou ERPs de grande porte.

O foco do escopo é:

- Processo de venda
- Cadastro de produtos
- Registro de transações
- Persistência de dados local ou remota
- Base sólida para extensões futuras

Funcionalidades adicionais só são consideradas se não comprometerem a leveza, a portabilidade e a clareza do sistema.

## Público-alvo

O projeto é voltado principalmente para:

- Pequenos comerciantes que precisam de uma solução simples e confiável
- Desenvolvedores que desejam adaptar ou integrar um PDV a outros sistemas
- Ambientes educacionais ou experimentais

Apesar disso, a arquitetura foi pensada de forma genérica o suficiente para **escalar e atender redes maiores**, caso haja interesse em evoluir o sistema.

## Filosofia do Projeto

Os princípios que orientam o desenvolvimento são:

- Baixo consumo de recursos como requisito
- Independência de plataformas proprietárias
- Simplicidade acima de abstrações desnecessárias
- Código compreensível e modificável
- Controle total dos dados pelo usuário

## Estado Atual e Visão

Atualmente, o projeto encontra-se em desenvolvimento ativo, com foco na consolidação do núcleo do PDV.

A visão de longo prazo é manter um **core enxuto**, permitindo que funcionalidades mais complexas sejam adicionadas de forma modular, sem descaracterizar o objetivo original.
